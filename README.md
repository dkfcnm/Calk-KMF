# Chinese Metaphysics Calculator

## О проекте

Chinese Metaphysics Calculator - это полнофункциональное приложение для работы с китайской метафизикой, включающее:
- Календарь Тун Шу (с различными представлениями: год, месяц, неделя, день, час)
- Персонализация по профилю пользователя (BaZi birth chart, 10 Богов, фазы Ци, скрытые стволы)
- Расклады Ци Мэнь с выбором разных методологий
- CRM-систему для работы с клиентами

## Системные требования

- Windows 10 или новее
- Python 3.8 или выше
- Node.js 14 или выше
- npm 6 или выше
- 4GB RAM (рекомендуется)
- 500MB свободного дискового пространства

## Установка

### Автоматическая установка зависимостей

1. Запустите `setup_deps.bat` для автоматической установки и настройки всех зависимостей
2. Следуйте инструкциям на экране
3. После успешной установки всех зависимостей, скрипт завершит работу

### Ручная установка

#### Python и необходимые библиотеки:
1. Установите Python с [официального сайта](https://www.python.org/downloads/)
2. Убедитесь, что отмечена опция "Add Python to PATH" при установке
3. Откройте командную строку и установите необходимые библиотеки:
   ```
   pip install fastapi uvicorn sqlalchemy psycopg2-binary python-dotenv
   ```

#### Node.js и npm:
1. Установите Node.js с [официального сайта](https://nodejs.org/)
2. Убедитесь, что npm установлен вместе с Node.js
3. Перейдите в директорию frontend и установите зависимости:
   ```
   cd frontend
   npm install
   ```

## Запуск и использование приложения

### Запуск приложения
1. Запустите `start_app.bat` для одновременного запуска бэкенда и фронтенда
2. Скрипт проверит наличие всех зависимостей и предложит установить отсутствующие
3. После успешного запуска, приложение будет доступно по адресам:
   - Фронтенд: http://localhost:3000
   - Бэкенд API: http://localhost:8000
   - Документация API: http://localhost:8000/docs

### Остановка приложения
1. Запустите `stop_app.bat` для корректного завершения работы приложения
2. Скрипт найдет и завершит все процессы, связанные с приложением

## Основные функциональные возможности

### Календарь Тун Шу
- Просмотр календаря на год / месяц / неделю / день
- Детальная информация по каждому часу выбранного дня
- Информация о благоприятных и неблагоприятных днях

### Расклады Ци Мэнь
- Выбор различных методологий расчета (Чжи Жунь, Чай Бу)
- Просмотр текущего расклада
- Расчет расклада на выбранную дату
- Детальная информация по дворцам

### Профили и персонализация (Phase 2)
- Создание профилей с датой/временем/городом рождения
- Автодополнение городов с timezone (104 города)
- Автоматический расчёт карты рождения (BaZi — 4 столба)
- Персонализированный календарь: 10 Богов, фазы Ци, скрытые стволы относительно day_master профиля

### CRM-система
- Управление базой клиентов
- Ведение истории сессий
- Связывание расчетов с клиентами
- Добавление заметок

## Устранение неполадок

### Ошибка "Node.js не установлен или отсутствует в PATH"
1. Убедитесь, что Node.js установлен
2. Проверьте, что путь к Node.js добавлен в переменную PATH
3. Перезагрузите компьютер после установки Node.js
4. Запустите `setup_deps.bat` для автоматической настройки

### Ошибка "Python не установлен или отсутствует в PATH"
1. Установите Python, отметив галочку "Add Python to PATH"
2. Перезагрузите компьютер
3. Запустите `setup_deps.bat` для автоматической настройки

### Ошибка "Порт 8000 уже занят"
1. Убедитесь, что предыдущая копия приложения не запущена
2. Запустите `stop_app.bat` для завершения всех процессов
3. Проверьте, какой процесс использует порт 8000:
   ```
   netstat -ano | findstr :8000
   ```
4. Завершите процесс, занимающий порт

### Проблемы при установке зависимостей npm
1. Удалите директорию `node_modules` в папке frontend
2. Запустите установку зависимостей заново:
   ```
   cd frontend
   npm install
   ```
3. Если проблемы сохраняются, проверьте журналы npm:
   ```
   npm cache clean --force
   npm install --verbose
   ```

## MCP интеграция (Model Context Protocol)

Проект поддерживает работу с БД PostgreSQL через MCP сервер для интеграции с IDE (VS Code, Roo Code и др.).

### Конфигурация MCP
- **Глобальная:** `~/.kimi/mcp.json` — конфигурация для Kimi Code CLI
- **Проектная:** `.roo/mcp.json` — конфигурация для Roo Code (VS Code)

### Доступные MCP серверы
- **postgres:** `@modelcontextprotocol/server-postgres` — read-only SQL запросы к `calk_kmf`
- **context7:** `@upstash/context7-mcp` — документация библиотек
- **playwright:** `@playwright/mcp` — автоматизация браузера

### Пример использования
MCP сервер предоставляет инструмент `query` для выполнения SQL запросов:
```json
{
  "name": "query",
  "description": "Run a read-only SQL query"
}
```

## База данных

**Архитектура:**
- **Опорная (основная) СУБД:** PostgreSQL 18 — хранит все справочники, результаты расчётов, аналитику и исторические данные.
- **Вспомогательная (offline/fallback) СУБД:** SQLite 3 — облегчает работу портала, используется для агрегатных представлений (например, `t_tung_shu_daily`) и быстрого доступа к данным без необходимости подключения к PostgreSQL.

- **База:** `calk_kmf`
- **Драйвер PostgreSQL:** pg8000 (Pure Python)
- **Период данных:** 2025-2027 гг.
- **Часовые пояса:** UTC-12 ... UTC+14

### Управление PostgreSQL

В папке `bat/` находятся скрипты для запуска и остановки сервера PostgreSQL (требуются права администратора):

- `bat\DB_Start.bat` — запускает службу PostgreSQL (`net start postgresql-x64-18`)
- `bat\DB_Stop.bat` — останавливает службу PostgreSQL (`net stop postgresql-x64-18`)

> **Примечание:** если у вас другая версия PostgreSQL, отредактируйте имя службы в этих файлах.
