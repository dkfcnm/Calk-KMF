# Технический контекст

## Стек технологий
- **Основной язык:** Python
- **База данных:** PostgreSQL 16+
- **Драйвер БД:** pg8000 (Pure Python)
- **ORM:** SQLAlchemy
- **Система миграций:** Alembic
- **Форматирование кода:** PEP 8, Ruff

## Архитектура проекта
Проект структурирован по модулям, соответствующим различным областям китайской метафизики:

```
code/
  ├── analysis/         - Модуль аналитики и обработки правил (Rule Engine)
  ├── bazi_calendar/    - Модуль расчета Бацзы и календарей
  ├── calendar/         - Модуль синхронизации с календарями
  ├── common/           - Общие компоненты (DBManager, Config)
  ├── feng_shui/        - Модуль расчетов Фэн-шуй (Летящие звезды)
  ├── qimen/            - Модуль расчетов Цимэнь (Чжи Рэн, Чай Бу)
  ├── taiyi/            - Модуль расчетов системы Тай И
  ├── tests/            - Регрессионные тесты
  └── utils/            - Утилиты для миграции и отладки
```

## База данных
- **СУБД:** PostgreSQL 16+ (Доступ строго через MCP сервер)
- **Размер:** ~5.5 ГБ
- **Таблиц:** 74 (24 основных, 10 контрольных, 37 справочных, 3 служебных)
- **Период данных:** 2025-2027 гг.
- **Записей:** ~350K в t_bazi_hourly, ~2M в t_qimen_dgiren_hourly
- **Шаблоны Ци Мэнь:** 1080 уникальных карт
- **Режим WAL:** Параллельные чтения во время записи
- **Пакетная обработка:** 50K записей за операцию

## Стратегия оптимизации
- **SQL-First:** Массовые операции выполняются на стороне БД, Python для оркестрации
- **Иерархичность:** Правила классифицируются по временному масштабу (Year, Month, Day, Hour)
- **Batch-Processing:** Чтение и запись данных только пакетами, избегая циклов Python
- **Шаблоны Ци Мэнь:** Замена вычислений поиском по шаблонам

## Принципы разработки
1. **DRY (Don't Repeat Yourself)** - минимизация дублирования кода и данных
2. **Модульная архитектура** - независимость компонентов для простоты тестирования
3. **Строгая типизация** - четкое определение структуры данных
4. **Документирование** - подробные docstrings на русском языке
5. **Оптимизация производительности** - эффективные алгоритмы для больших объемов данных

## Качество данных
- **Регрессионное тестирование:** 10 таблиц с эталонными данными
- **Контрольные точки:** Сверка с данными Гонконгской обсерватории и эталонами
- **Автоматическая проверка:** После каждого изменения (100% покрытие)
