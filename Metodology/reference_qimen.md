## Алгоритм расчета (Чай Бу)

### 1. Определение Инь/Ян Дунь
Основано на Солнечном сезоне (`solar_term_id`):
- **Ян Дунь (Yang)**: Сезоны 22-24, 1-9 (Зимнее солнцестояние -> ... -> Малое изобилие).
- **Инь Дунь (Yin)**: Сезоны 10-21 (Летнее солнцестояние -> ... -> Большие снега).

### 2. Определение Юаня (Yuan)
Используется индекс дня в цикле 60 Цзя-Цзы:
- `Yuan Index = (Day_Cycle_Index // 5) % 3`
- 0: Верхний Юань (Upper)
- 1: Средний Юань (Middle)
- 2: Нижний Юань (Lower)

### 3. Определение Номера Расклада (Ju Number)
По таблице соответствия `Сезон -> (Верхний, Средний, Нижний)`.

### 4. Земная Тарелка (Earth Plate)
Размещение 6 Инструментов и 3 Непарных (Wu, Ji, Geng, Xin, Ren, Gui, Ding, Bing, Yi) по дворцам:
- **Yang Dun**: Прямой порядок (1 -> 2 -> ... -> 9 -> 1).
- **Yin Dun**: Обратный порядок (9 -> 8 -> ... -> 1 -> 9).
Старт (Wu) находится во дворце, равном номеру расклада (`Ju Number`).

### 5. Лидер (Xun Shou)
Определяется по Небесному стволу и Земной ветви часа (60 Цзя-Цзы).
Лидер — это ствол (Wu, Ji, Geng, Xin, Ren, Gui), управляющий текущей декадой.

### 6. Главная Звезда (Zhi Fu) и Врата (Zhi Shi)
- Находятся во дворце, где расположен Лидер на Земной тарелке.
- Если Лидер попадает в центр (5), он приравнивается к дворцу 2 (Кунь).

### 7. Небесная Тарелка (Heaven Plate)
- Главная Звезда (Zhi Fu) перемещается во дворец, где находится Небесный ствол часа на Земной тарелке.
    - *Исключение*: Если час Цзя (Jia), используется ствол Лидера.
- Остальные звезды сохраняют относительный порядок (всегда по часовой стрелке).
- Вместе со звездой перемещается Небесный ствол (с Земной тарелки исходного дворца).

### 8. Врата (Gate Plate)
- Главные Врата (Zhi Shi) перемещаются во дворец, соответствующий Ветви часа.
- Движение по дворцам:
    - **Yang Dun**: 1 -> 2 -> ... -> 9
    - **Yin Dun**: 9 -> 8 -> ... -> 1
- Остальные врата выстраиваются по кольцу (по часовой стрелке).

### 9. Духи (God Plate)
- Главный Дух (Чжи Фу) располагается там же, где Главная Звезда.
- Остальные духи выстраиваются по кольцу:
    - **Yang Dun**: По часовой стрелке.
    - **Yin Dun**: Против часовой стрелки.

## 3. Оптимизированная структура таблиц (Template Library v2)

# Оптимизированная структура таблиц Ци Мэнь Дунь Цзя (Template Library v2)

## Основной принцип: Библиотека Шаблонов
Для радикального уменьшения объема базы данных и ускорения работы используется подход "Библиотека Шаблонов".
Вместо хранения полной информации о звездах, вратах и духах для каждой записи (миллионы строк), таблицы фактов хранят только ссылку (`rasklad_id`) на шаблон.

### Таблица шаблонов: `spr_qimen_templates`
Содержит все 1080 уникальных комбинаций карт Ци Мэнь (18 раскладов * 60 часов * 9 дворцов = теоретический максимум, реально 1080 уникальных "Ju + Pillar").
- **PK**: `rasklad_id`, `palace_no`
- **Поля**:
  - `rasklad_id`: Текстовый ID вида `{Yin/Yang}_{Ju}_{Stem}{Branch}` (например, `Yang_1_甲子`).
  - `palace_no`: Номер дворца (1-9).
  - `heaven_stem`: Небесный ствол (Небесная тарелка).
  - `earth_stem`: Земной ствол (Земная тарелка).
  - `star`: Звезда.
  - `gate`: Врата.
  - `spirit`: Дух.

---

## Таблицы Фактов (Lean Schema)

### Метод Джи Рэн (Zhi Run)

#### 1. Часовые расклады (`t_qumen_dgiren_hourly`)
- **PK**: `chart_id`
- **Поля**:
  - `chart_id`: Уникальный хеш.
  - `hour_id`: Ссылка на час (`v_bazi_hourly`).
  - `rasklad_id`: Ссылка на шаблон (`spr_qimen_templates`).
  - `palace_no`: Номер дворца (1-9).
  - `chart_type`: 'Часовой'.
- **Логика расчета**:
  - Используется метод Фу Тоу (смещение).
  - **Важно**: Если день Фу Тоу совпадает с днем смены солнечного сезона, то **весь день** (и вся пятидневка) относятся к **новому** сезону (проверка сезона на конец дня 23:59:59).

#### 2. Дневные, Месячные, Годовые
- Таблицы: `t_qumen_dgiren_day`, `t_qumen_dgiren_month`, `t_qumen_dgiren_year`.
- Структура аналогична (хранят ключи периода + `rasklad_id` + `palace_no`).

### Метод Чай Бу (Chai Bu)

#### 1. Часовые расклады (`t_qumen_chauby_hourly`)
- **PK**: `chart_id`
- **Поля**:
  - `chart_id`: Уникальный хеш.
  - `hour_id`: Ссылка на час (`v_bazi_hourly`).
  - `rasklad_id`: Ссылка на шаблон (`spr_qimen_templates`).
  - `palace_no`: Номер дворца (1-9).
  - `chart_type`: 'Часовой'.
- **Логика расчета**:
  - Строго по текущему солнечному сезону на момент часа.
  - Не использует смещение Фу Тоу.

#### 2. Дневные, Месячные, Годовые
- Таблицы: `t_qumen_chauby_day`, `t_qumen_chauby_month`, `t_qumen_chauby_year`.
- Используют соответствующую логику (Ri Jia для дня и т.д.).

## Получение полных данных (SQL)
Для анализа или отображения необходимо делать JOIN с таблицей шаблонов:

```sql
SELECT 
    t.hour_id,
    t.palace_no,
    tmpl.heaven_stem,
    tmpl.earth_stem,
    tmpl.star,
    tmpl.gate,
    tmpl.spirit
FROM t_qumen_dgiren_hourly t
JOIN spr_qimen_templates tmpl 
    ON t.rasklad_id = tmpl.rasklad_id 
    AND t.palace_no = tmpl.palace_no;
```
