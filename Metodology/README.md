# Metodology — Методология проекта Calk_KMF

**Назначение:** Единая точка входа для понимания архитектуры, методологий, справочников и планов проекта.

> **⚠️ Правило обновления:** Обновлять только **единые файлы**, перечисленные ниже. Исходные разрозненные файлы перемещены в `archive/`. При необходимости правки — вносите изменения в единые справочники, а не в архив.

---

## Структура папки

### Опорные файлы (читать в первую очередь)

| Файл | Описание | Когда читать |
|------|----------|-------------|
| `../SESSION_CONTEXT.md` | **Текущий фокус, риски, правила** | В начале КАЖДОЙ сессии |
| `../PROJECT.md` | Главный живой документ: цели, стек, статус фаз | Когда нужен общий статус |
| `PROJECT_IMPLEMENTATION_STATUS.md` | Полный статус реализации: модули, API, тесты, бэклог | Когда нужна детализация по модулю |
| `00_project_architecture_v2.md` | Архитектурные решения: Rolling Window, SQL-First, MCP | При разработке нового модуля |
| `ai_collaboration_protocol.md` | Протокол взаимодействия с AI-агентами | При сомнениях в процессе работы |
| `technical_standards.md` | Технические стандарты разработки | При code review или спорах о реализации |

## Частые сценарии (куда идти за чем)

### «Нужно добавить новое правило анализа»
1. `SESSION_CONTEXT.md` → проверить риски
2. `04_analysis_module_spec.md` → спецификация Rule Engine
3. `00_project_architecture_v2.md` → SQL-First, иерархия
4. `reference_tongshu_methodologies.md` → методология правила
5. `postgresql_schema.md` → структура `t_rule_registry`

### «Нужно разобраться в Ци Мэнь»
1. `reference_qimen.md` → полный справочник
2. `code/qimen/` → код

### «Что-то сломалось в БД»
1. `SESSION_CONTEXT.md` → активные риски
2. `bz.md` → известные проблемы
3. `postgresql_schema.md` → структура таблиц
4. `data/ddl_full_schema_raw.sql` → DDL для сравнения

### «Нужно начать новый модуль»
1. `SESSION_CONTEXT.md` → текущий фокус
2. `00_project_architecture_v2.md` → архитектурные паттерны
3. `technical_standards.md` → стандарты кода
4. `PROJECT.md` → раздел 7 (фазы) — куда вписать
