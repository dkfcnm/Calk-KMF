# Схема таблицы t_qumen_dgiren (Ци Мэнь Дунь Цзя - Джи Рэн)

Таблица предназначена для хранения рассчитанных раскладов Ци Мэнь Дунь Цзя по методу Джи Рэн (Zhi Run / Intercalation).
Структура идентична таблице `t_qumen_chauby`.

## DDL

```sql
CREATE TABLE IF NOT EXISTS t_qumen_dgiren (
    hour_id TEXT NOT NULL,              -- Ссылка на час в t_bazi_hourly
    chart_num INTEGER,                  -- Номер расклада (Ju Number)
    yin_yang TEXT,                      -- Тип Дунь: 'Yin' или 'Yang'
    chart_type TEXT,                    -- Тип карты (пока только 'Часовой')
    palace_no INTEGER,                  -- Номер дворца (1-9) по Ло Шу
    earth_stem TEXT,                    -- Небесный ствол на Земной тарелке
    is_fou_tou_earth INTEGER,           -- Флаг: является ли ствол лидером (Fou Tou) на Земле (1/0)
    heaven_stem TEXT,                   -- Небесный ствол на Небесной тарелке
    is_fou_tou_heaven INTEGER,          -- Флаг: является ли ствол лидером (Fou Tou) на Небе (1/0)
    star TEXT,                          -- Звезда (сокращенный иероглиф)
    is_main_star INTEGER,               -- Флаг: является ли главной звездой (Zhi Fu) (1/0)
    gate TEXT,                          -- Врата (сокращенный иероглиф)
    is_main_gate INTEGER,               -- Флаг: являются ли главными вратами (Zhi Shi) (1/0)
    spirit TEXT,                        -- Дух (сокращенный иероглиф)
    PRIMARY KEY (hour_id, palace_no)
);
```

## Отличия метода Джи Рэн (Zhi Run)

Основное отличие от Чай Бу (Chai Bu) заключается в выборе Солнечного сезона (Solar Term) для определения номера расклада (Ju).

1.  **Фу Тоу (Fu Tou / Head)**: Определяется "Глава" текущей пятидневки (Jia или Ji).
    - Если НС дня 甲(1) или 己(6) -> Фу Тоу сегодня.
    - Если НС дня 乙(2) или 庚(7) -> Фу Тоу был вчера.
    - И т.д.
2.  **Выбор Сезона**: Используется Солнечный сезон, в который попадает **день Фу Тоу**, а не текущий день.
    - Это реализует принцип "Чао Шэнь" (Сверхдух), когда энергия новой пятидневки (Духа) приходит раньше смены сезона (Ци).
    - В этом случае расклад переключается на следующий сезон раньше астрономического срока.
    - В редких случаях (Цзе Ци) Дух опаздывает, и расклад задерживается.
3.  **Вставки (Intercalation)**: При накоплении расхождения более 9-10 дней производится вставка (повторение) сезона. В данной реализации используется упрощенный алгоритм "Сезон по Фу Тоу", который автоматически учитывает большинство сдвигов, но полная вставка (Run Qi) требует анализа годового масштаба.
