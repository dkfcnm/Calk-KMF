# Дорожная карта развития бацзы-календаря до уровня Тун Шу

Обновлено: 2025-12-25 16:10 (MSK)

## 1. Общая структура пакета `code.bazi_calendar`

- `db` — инфраструктура работы с SQLite (`calk_kmf.sqlite`):
  - подключение (`get_sqlite_connection`),
  - резервное копирование (`backup_sqlite_db`),
  - утилиты работы с `datetime` в UTC (`ensure_utc`, `parse_sqlite_timestamp`).
- `solar_terms` — логика работы с солнечными сезонами:
  - загрузка значений из `t_solar_term_time` и `t_solar_term_time_hko`,
  - приоритет данных HKO и fallback на расчётные значения,
  - функции `get_prev_solar_term`, `get_solar_term_crossing`, `tz_from_offset`.
- `pillars` — расчёт столпов:
  - `calc_year_pillar`, `calc_month_pillar`, `calc_day_pillar`, `calc_hour_pillar`,
  - агрегирующая функция `calc_four_pillars`,
  - базовые структуры `Pillar`, `FourPillars`.
- `hourly` — слой хранения бацзы-календаря:
  - `ensure_bazi_hourly_table`,
  - `populate_bazi_hourly_table`, `populate_bazi_hourly_for_years`,
  - генерация `v_bazi_hourly`, `v_bazi_hourly_tz_*`, `v_bazi_hourly_msk`.
- Вспомогательные утилиты:
  - `analyze_mismatches.py` — диагностика контрольных расхождений,
  - `rebuild_control_data.py` — пересборка контрольных столпов.

## 2. План развития до уровня Тун Шу

1. **Методология Тун Шу**
   - собрать перечень разделов (офицеры дня, благоприятные/неблагоприятные действия, 28 созвездий, звёзды, небесные божества, сочетания, индикаторы и т.д.);
   - задокументировать источники правил и точность (астрономические события, лунно‑солнечный календарь).

2. **Справочники и данные**
   - расширить SQLite новыми таблицами `spr_*` для сущностей Тун Шу;
   - добавить расчётные таблицы (например, `t_moon_phase`, `t_tongshu_daily`, `t_tongshu_hourly`);
   - подготовить скрипты наполнения и контрольные наборы.

3. **Расчётный слой `code.bazi_calendar.tongshu`**
   - создать подпакет `tongshu/`;
   - реализовать чистые функции для каждой сущности (офицеры, индикаторы, благоприятность действий и т.д.);
   - ввести агрегирующий объект, формирующий “профиль дня/часа” Тун Шу на основе бацзы и дополнительных данных.

4. **Хранилище и представления**
   - расширить `t_bazi_hourly` или создать `t_tongshu_hourly`/`t_tongshu_daily`;
   - реализовать массовое заполнение (по аналогии с `populate_bazi_hourly_for_years`);
   - создать view `v_tongshu_hourly`, `v_tongshu_tz_*`, `v_tongshu_msk`.

5. **Валидация и контроль**
   - сформировать контрольные значения Тун Шу (CSV/SQLite);
   - разработать скрипт проверки (`check_tongshu.py`), добившись 100% совпадения с эталоном;
   - интегрировать в процесс пересборки (`rebuild_control_data.py` дополнить или добавить новый инструмент).

6. **Документация и журнал**
   - обновить методологию: добавить разделы по правилам Тун Шу, схемам таблиц, описанию алгоритмов;
   - вести журнал изменений (project log) с фиксацией ключевых решений, источников данных и результатов проверок.

## 3. Принципы и требования

- **Хранилище:** все данные и расчёты — в SQLite, относительные пути внутри репозитория.
- **Приоритет наблюдений:** для солнечных терминов используется `t_solar_term_time_hko`, при отсутствии — `t_solar_term_time`.
- **Автономность:** справочники и расчёты не должны зависеть от внешних сервисов в рантайме.
- **Контроль качества:** контрольные данные поддерживаются в актуальном виде, тесты (`check_control.py`, будущий `check_tongshu.py`) показывают 0 ошибок до начала следующего этапа.
- **Документация:** каждое изменение схемы, алгоритма или структуры фиксируется в соответствующих файлах методологии.
