# Оптимизированная структура таблиц Ци Мэнь Дунь Цзя (Template Library v2)

## Основной принцип: Библиотека Шаблонов
Для радикального уменьшения объема базы данных и ускорения работы используется подход "Библиотека Шаблонов".
Вместо хранения полной информации о звездах, вратах и духах для каждой записи (миллионы строк), таблицы фактов хранят только ссылку (`rasklad_id`) на шаблон.

### Таблица шаблонов: `spr_qimen_templates`
Содержит все 1080 уникальных комбинаций карт Ци Мэнь (18 раскладов * 60 часов * 9 дворцов = теоретический максимум, реально 1080 уникальных "Ju + Pillar").
- **PK**: `rasklad_id`, `palace_no`
- **Поля**:
  - `rasklad_id`: Текстовый ID вида `{Yin/Yang}_{Ju}_{Stem}{Branch}` (например, `Yang_1_甲子`).
  - `palace_no`: Номер дворца (1-9).
  - `heaven_stem`: Небесный ствол (Небесная тарелка).
  - `earth_stem`: Земной ствол (Земная тарелка).
  - `star`: Звезда.
  - `gate`: Врата.
  - `spirit`: Дух.

## Таблицы Фактов (Lean Schema)

### Метод Джи Рэн (Zhi Run)

#### 1. Часовые расклады (`t_qumen_dgiren_hourly`)
- **PK**: `chart_id`
- **Поля**:
  - `chart_id`: Уникальный хеш.
  - `hour_id`: Ссылка на час (`v_bazi_hourly`).
  - `rasklad_id`: Ссылка на шаблон (`spr_qimen_templates`).
  - `palace_no`: Номер дворца (1-9).
  - `chart_type`: 'Часовой'.
- **Логика расчета**:
  - Используется метод Фу Тоу (смещение).
  - **Важно**: Если день Фу Тоу совпадает с днем смены солнечного сезона, то **весь день** (и вся пятидневка) относятся к **новому** сезону (проверка сезона на конец дня 23:59:59).

#### 2. Дневные, Месячные, Годовые
- Таблицы: `t_qumen_dgiren_day`, `t_qumen_dgiren_month`, `t_qumen_dgiren_year`.
- Структура аналогична (хранят ключи периода + `rasklad_id` + `palace_no`).

### Метод Чай Бу (Chai Bu)

#### 1. Часовые расклады (`t_qumen_chauby_hourly`)
- **PK**: `chart_id`
- **Поля**:
  - `chart_id`: Уникальный хеш.
  - `hour_id`: Ссылка на час (`v_bazi_hourly`).
  - `rasklad_id`: Ссылка на шаблон (`spr_qimen_templates`).
  - `palace_no`: Номер дворца (1-9).
  - `chart_type`: 'Часовой'.
- **Логика расчета**:
  - Строго по текущему солнечному сезону на момент часа.
  - Не использует смещение Фу Тоу.

#### 2. Дневные, Месячные, Годовые
- Таблицы: `t_qumen_chauby_day`, `t_qumen_chauby_month`, `t_qumen_chauby_year`.
- Используют соответствующую логику (Ri Jia для дня и т.д.).

## Получение полных данных (SQL)
Для анализа или отображения необходимо делать JOIN с таблицей шаблонов:

```sql
SELECT 
    t.hour_id,
    t.palace_no,
    tmpl.heaven_stem,
    tmpl.earth_stem,
    tmpl.star,
    tmpl.gate,
    tmpl.spirit
FROM t_qumen_dgiren_hourly t
JOIN spr_qimen_templates tmpl 
    ON t.rasklad_id = tmpl.rasklad_id 
    AND t.palace_no = tmpl.palace_no;
```
