## Фактическая реализация бацзы-календаря

Ниже зафиксирована текущая техническая реализация требований данного документа.

### База данных и часовой пояс

- Расчёты ведутся в базе **SQLite** `calk_kmf.sqlite` в корне проекта.
- Все расчётные таблицы хранят время в **UTC (0 часовой пояс)**.
- Привязка к локальным часовым поясам (Москва, Пекин и т.п.) выполняется **только во VIEW** с использованием функций SQLite `datetime(..., '+N hours')`.

### Расчётная таблица часового календаря

- Основная расчётная таблица часового календаря: `t_bazi_hourly`.
- Таблица заполняется функцией `populate_bazi_hourly_table` из модуля `code/bazi_calendar/bazi_calendar.py`.
- Таблица хранит:
  - `datetime_utc` — момент начала часа в формате ISO, в UTC;
  - день недели (кратко по-русски);
  - номер сезона (`solar_term_index`, совпадает с `spr_solar_term.solar_term_id`);
  - ветвь часа и все четыре столпа (год/месяц/день/час) с разложением на ствол/ветвь.
- При смене сезона внутри часа в таблицу попадает **две строки** для этого часа (до и после смены), что соответствует требованию выше.

Подробная структура столбцов описана в методологическом файле `bazci_04_schema_bazi_hourly.md`.

### Представления для чтения результатов

1. **`v_bazi_hourly` (UTC)**

   - Базовое представление часового календаря в UTC.
   - Текстовые названия подтягиваются из справочников:
     - название сезона — из `spr_solar_term.solar_term_name_ru` по `solar_term_id`;
     - название часа — из `spr_earthly_branch.branch_rus` по ветви часа.
   - Содержит полный набор колонок, перечисленных в задаче (дата/время, день недели, сезон, столпы и их разложение).

2. **`v_bazi_hourly_msk` (Москва, UTC+3)**

   - Представление для московского времени строится **поверх `v_bazi_hourly`**.
   - Поле `datetime_msk` = `datetime(datetime_utc, '+3 hours')`.
   - Остальные колонки полностью соответствуют `v_bazi_hourly`.

Все изменения структуры `t_bazi_hourly` и связанных VIEW должны сопровождаться обновлением файла `Metodology/bazci_04_schema_bazi_hourly.md`.
