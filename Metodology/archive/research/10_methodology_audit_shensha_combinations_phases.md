## 3. Проблемы и риски

| ID | Проблема | Влияние | Решение |
|----|----------|---------|---------|
| P.1 | `spr_tongshu_shensha_rule` пустая | Невозможно рассчитать Symbolic Stars | Наполнить данными из `bazci_star.md` |

| P.5 | `TongShuDay` не считает звёзды | Нет Symbolic Stars в выводе | Добавить метод `_load_symbolic_stars()` |

| P.7 | `TongShuDay` не считает фазы Ци | Нет Qi Phases в выводе | Добавить метод `_load_qi_phases()` |
| P.8 | Нет предикатов в Rule Engine | Комбинации не участвуют в скоринге | Создать предикаты `CHECK_BRANCH_COMBO`, `CHECK_STEM_COMBO` |

## 4. Рекомендуемый план реализации

### 4.1 Этап 5.12 — Наполнение справочников

1. **spr_tongshu_ten_god:** Скрипт наполнения из `tungshu_10_gods.md` (100 записей: 10 Day Masters × 10 Gods)
2. **spr_tongshu_phase + phase_mapping:** Скрипт наполнения из `bazci_faza_ci.md` (12 фаз × 10 стволов)
3. **spr_tongshu_shensha_rule:** Скрипт наполнения из `bazci_star.md` (~150+ правил)

### 4.2 Этап 2.X — Реализация в TongShuDay

1. **`_load_ten_gods(day_stem)`** — загрузка 10 богов для каждого столпа относительно Day Master
2. **`_load_symbolic_stars()`** — поиск всех звёзд по правилам из `spr_tongshu_shensha_rule`
3. **`_load_qi_phases()`** — определение фазы Ци для каждого столпа
4. **`_calculate_combinations()`** — проверка всех пар столпов на комбинации

### 4.3 Этап Rule Engine

1. **`CHECK_BRANCH_COMBO`** — предикат для комбинаций земных ветвей
2. **`CHECK_STEM_COMBO`** — предикат для комбинаций небесных стволов
3. **`CHECK_SHENSHA`** — предикат для символических звёзд
